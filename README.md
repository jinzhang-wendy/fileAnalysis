# 📄 AI 智能文档分析工具

> 一个基于人工智能的文档分析工具，让你的文档"开口说话"

---

## 🎯 这个工具能做什么？

想象一下，你有一份 100 页的产品说明书，你想知道：
- "这个产品的保修期是多久？"
- "退货流程是怎样的？"
- "主要功能有哪些？"

**传统方式**：你需要一页一页翻阅、搜索关键词，可能要花上十几分钟。

**使用本工具**：
1. 上传文档
2. 直接提问
3. 秒速得到答案（还会告诉你答案在文档的哪个位置）

---

## ✨ 核心功能

### 1. 📤 智能文档解析
支持多种常见文档格式，自动提取文字内容：

| 格式 | 说明 |
|------|------|
| `.txt` | 纯文本文件 |
| `.md` | Markdown 文档 |
| `.pdf` | PDF 文档 |
| `.docx` | Word 文档 |
| `.xlsx` / `.xls` | Excel 表格 |
| `.csv` | CSV 数据文件 |

### 2. 🔍 四种分析模式

#### 模式一：智能识别（推荐）
让 AI 自动判断文档类型，选择最合适的分析方式：
- 流程类文档 → 生成流程图
- 报告论文 → 生成摘要和要点
- 合同规范 → 提取关键条款和实体

#### 模式二：流程图生成
将文档中的流程描述转换成可视化流程图，一目了然。

#### 模式三：内容摘要
自动生成文档的：
- 标题
- 核心摘要（200字以内）
- 关键要点列表
- 关键词标签

#### 模式四：结构分析
分析文档的层级结构：
- 章节划分
- 各章节摘要
- 关键实体提取（人物、组织、日期、金额等）

### 3. 💬 智能问答（核心功能）

这是本工具最强大的功能。你可以像和人对话一样，向文档提问：

```
用户：这个产品的保修期是多久？
AI：根据文档第 3 页第 2 段，本产品保修期为 2 年，自购买之日起计算。

用户：那如果坏了怎么办？
AI：根据之前提到的保修期和文档第 5 页的售后流程，您需要：
    1. 联系客服热线 400-xxx-xxxx
    2. 提供购买凭证
    3. 寄回产品进行检测...
```

#### 什么是"长短记忆联动"？

我们模拟了人类的记忆方式：

| 记忆类型 | 类比人类 | 在工具中 |
|---------|---------|---------|
| **长期记忆** | 你学过的知识、看过的书 | 文档内容（通过 RAG 技术检索） |
| **短期记忆** | 刚才聊天的内容 | 最近几轮对话记录 |

当你追问"那如果坏了怎么办"时，工具会：
1. **长期记忆**：从文档中检索"售后"、"维修"相关内容
2. **短期记忆**：结合之前提到的"保修期"，理解你在问保修期内的维修流程
3. **综合回答**：给出完整的解决方案

### 4. 🛡️ 稳健的错误处理

- **网络超时**：自动重试，最多等待 30 秒
- **请求失败**：智能重试，间隔递增（1秒→2秒→4秒）
- **随时取消**：点击"取消"按钮中断正在进行的操作
- **友好提示**：错误发生时告诉你具体原因和解决方法

### 5. ⚡ 性能优化特性

#### 智能缓存
- 文档解析结果自动缓存，相同文档无需重复处理
- 向量嵌入数据缓存，节省 API 调用和时间
- 缓存自动过期清理（默认 24 小时）

#### 请求队列管理
- 自动控制并发请求数量，避免触发 API 限流
- 速率限制保护（每分钟最多 30 个请求）
- 请求优先级排序，重要请求优先处理

#### 异步处理
- 大型任务分片执行，界面始终保持响应
- 实时进度报告，清晰了解处理状态
- 支持随时取消正在进行的任务

### 6. 💰 成本控制

#### Token 使用统计
- 实时记录每次 API 调用的 Token 消耗
- 精确估算中英文内容的 Token 数量
- 成本实时计算（支持 DeepSeek 定价）

#### 权限与限额
| 用户级别 | 每次最大 Token | 每日请求限制 | 单文件大小限制 |
|---------|---------------|-------------|---------------|
| Guest | 2,000 | 10 | 1 MB |
| User | 4,000 | 100 | 5 MB |
| Premium | 8,000 | 500 | 20 MB |
| Admin | 16,000 | 无限制 | 100 MB |

#### 预算保护
- 设置每日预算限制（默认 $10/天）
- 接近预算时自动告警
- 超出预算自动降级，避免超支

## 🚀 快速开始

### 第一步：安装 Node.js

本工具基于 Node.js 运行，需要先安装它。

**Windows 用户：**
1. 访问 [https://nodejs.org/](https://nodejs.org/)
2. 下载 LTS（长期支持版）安装包
3. 双击安装，一路"下一步"即可

**Mac 用户：**
```bash
# 如果已安装 Homebrew，直接运行：
brew install node

# 没有安装 Homebrew？访问 https://nodejs.org/ 下载安装包
```

**验证安装：**
打开终端/命令行，输入：
```bash
node -v
npm -v
```
如果显示版本号（如 v20.x.x），说明安装成功。

### 第二步：下载项目

```bash
# 方式一：使用 Git（推荐）
git clone https://github.com/your-username/ai-doc-analyzer.git
cd ai-doc-analyzer

# 方式二：直接下载 ZIP
# 1. 点击项目页面右上角的 "Code" → "Download ZIP"
# 2. 解压后进入项目文件夹
```

### 第三步：安装依赖

```bash
npm install
```

这一步会下载项目需要的所有第三方库，首次安装可能需要几分钟。

### 第四步：配置 API Key

本工具使用 DeepSeek 的 AI 服务，需要获取 API Key（类似一把"钥匙"）。

#### 如何获取 DeepSeek API Key：

1. 访问 [https://platform.deepseek.com/](https://platform.deepseek.com/)
2. 注册/登录账号
3. 进入"API Keys"页面
4. 点击"创建 API Key"
5. 复制生成的 Key（以 `sk-` 开头的一串字符）

#### 配置项目：

在项目根目录创建 `.env.local` 文件：

```bash
# 在项目文件夹中创建配置文件
# Windows（命令行）：
echo VITE_DEEPSEEK_API_KEY=sk-你的密钥 > .env.local

# Mac/Linux：
echo "VITE_DEEPSEEK_API_KEY=sk-你的密钥" > .env.local
```

或者用任何文本编辑器创建 `.env.local` 文件，内容如下：
```
VITE_DEEPSEEK_API_KEY=sk-xxxxxxxxxxxxxxxxxxxxxxxx
```

> ⚠️ **重要提示**：
> - 不要把 API Key 分享给他人
> - 不要把 `.env.local` 文件上传到公开的代码仓库
> - DeepSeek 新用户通常有免费额度，足够日常使用

### 第五步：启动项目

```bash
npm run dev
```

看到类似以下输出，说明启动成功：
```
VITE v7.x.x  ready in xxx ms

➜  Local:   http://localhost:5173/
➜  Network: use --host to expose
```

### 第六步：开始使用

1. 打开浏览器，访问 `http://localhost:5173`
2. 点击"选择文件"上传你的文档
3. 等待文档解析完成
4. 点击"建立索引"（首次问答前需要）
5. 开始提问或选择分析模式

## ❓ 常见问题（FAQ）

### Q1: 我不懂编程，能用这个工具吗？

**A:** 完全可以！只要你按照上面的"快速开始"步骤操作，都是简单的复制粘贴和点击。使用时更是完全图形化界面，零编程门槛。

### Q2: 为什么是 DeepSeek？能用 ChatGPT 吗？

**A:** DeepSeek 是国产 AI 服务，价格实惠、中文效果好、国内访问稳定。如果你想使用其他 AI 服务（如 ChatGPT、Claude），可以修改代码中的 API 地址和模型名称，但需要一定的编程知识。

### Q3: 我的文档包含敏感信息，安全吗？

**A:**
- 文档内容只会在你的电脑上处理
- 发送给 AI 的只是文字内容（不含文件名、元数据等）
- 不会保存到任何第三方服务器
- 建议不要上传涉及国家机密、核心商业机密的文档

### Q4: API 调用要花钱吗？

**A:**
- DeepSeek 新用户有免费额度
- 正常使用情况下，分析一份文档 + 多轮问答，费用通常不到 1 分钱
- 如果担心费用，可以在 DeepSeek 后台设置使用限额

### Q5: 为什么有时候回答不准确？

**A:** AI 不是万能的，以下情况可能导致回答不准确：
- 文档内容本身模糊或有歧义
- 问题太笼统，无法精准匹配
- 文档太短，信息不足

**建议**：
- 尽量问具体的问题
- 可以追问获得更准确的答案
- 对重要信息，最好对照原文核实

### Q6: 支持图片中的文字识别（OCR）吗？

**A:** 当前版本不支持。如果你的 PDF 是扫描件（图片形式），需要先用 OCR 工具转换成可搜索的 PDF 或文本文件。

### Q7: 能处理多大的文档？

**A:**
- 前端限制：单文件最大 10MB
- 实际处理能力取决于文档类型和内容密度
- 一般 100 页以内的文档都能流畅处理

## 🔧 技术原理（给好奇的你）

### 什么是 RAG？

RAG（Retrieval-Augmented Generation，检索增强生成）是一种让 AI "记住"大量信息的技术。

**通俗理解**：

想象你要参加一场开卷考试：
- **传统 AI**：只能靠"记忆"，考场上不能翻书，可能记不全
- **RAG**：允许带一本参考书，遇到问题先翻书找答案

我们的工具就是这样：
1. 你的文档就是那本"参考书"
2. 提问时，先从文档中检索相关段落
3. 把相关段落和问题一起发给 AI
4. AI 基于文档内容回答，答案更准确

### 长短记忆联动系统

我们实现了仿人类的记忆管理系统：

| 记忆层级 | 类比人类 | 在工具中 | 保留策略 |
|---------|---------|---------|---------|
| **核心记忆** | 永远记住的知识 | 重要对话、关键信息 | 永久保留（30% 容量） |
| **近期记忆** | 刚才说的话 | 最近几轮对话 | 按时间排序，满了才压缩 |
| **压缩记忆** | 大概印象 | 对话摘要 | 保留首尾，中间压缩 |
| **丢弃记忆** | 忘记的内容 | 不重要的信息 | 超出容量时丢弃 |

### 智能降级策略

当系统遇到压力时，会自动降级以保证基本服务：

| 降级级别 | 触发条件 | 系统行为 |
|---------|---------|---------|
| **Normal** | 正常运行 | 完整功能，无限制 |
| **Light** | 接近预算限制 | 减少 30% Token 使用 |
| **Medium** | 错误率 > 10% | 跳过向量检索，使用关键词匹配 |
| **Heavy** | 错误率 > 30% | 仅使用缓存，简化回答 |
| **Minimal** | 超出预算 | 最小化服务，简短回答 |

### 核心服务说明

#### 1. 缓存服务 (cacheService.ts)
- 自动缓存文档解析结果，避免重复处理
- 缓存向量嵌入数据，节省 API 调用
- 支持过期清理和存储空间管理（最大 50MB）
- 使用 localStorage 持久化存储

#### 2. 上下文管理器 (contextManager.ts)
- **记忆管理**：分级存储对话历史，智能压缩过期内容
- **Token 计数**：精确估算中英文 Token 数量
- **权限控制**：四级权限（guest/user/premium/admin），不同限额
- **成本追踪**：实时记录 Token 使用和费用
- **降级策略**：根据系统状态自动调整服务级别

#### 3. 请求队列 (requestQueue.ts)
- 并发控制：同时最多 3 个请求
- 速率限制：每分钟最多 30 个请求
- 优先级队列：重要请求优先处理
- 自动重试：网络错误自动重试（最多 3 次）

#### 4. 异步处理器 (asyncProcessor.ts)
- 分片处理大数据，避免阻塞 UI
- 支持任务取消
- 进度实时报告
- 并发限制执行

## 🛠️ 高级配置

### 自定义超时时间

默认 API 请求超时为 30 秒，可以在 `src/services/apiClient.ts` 中修改：

```typescript
const DEFAULT_CONFIG: ApiClientConfig = {
  defaultTimeout: 30000,  // 修改这里，单位毫秒
  maxRetries: 3,
  baseRetryDelay: 1000,
};
```

### 调整记忆容量

默认保留最近 3 轮对话（6 条消息），可以在 `src/services/ragService.ts` 中修改：

```typescript
const SHORT_TERM_MEMORY_LIMIT = 6;  // 修改这里，建议 4-10 之间
```

### 配置上下文管理器

在 `src/services/contextManager.ts` 中可以调整：

```typescript
const DEFAULT_CONFIG: ContextConfig = {
  maxContextTokens: 4000,          // 最大上下文 Token 数
  maxHistoryTurns: 10,             // 最多保留 10 轮对话
  coreMemoryRatio: 0.3,            // 30% 用于核心记忆
  compressionThreshold: 2000,      // 超过 2000 Token 触发压缩
  budgetLimit: 10,                 // 每天 $10 预算
};
```

### 配置请求队列

在 `src/services/requestQueue.ts` 中可以调整：

```typescript
const DEFAULT_CONFIG: QueueConfig = {
  maxConcurrent: 3,           // 同时最多 3 个请求
  rateLimitPerMinute: 30,     // 每分钟最多 30 个请求
  retryDelay: 1000,           // 重试延迟 1 秒
  maxRetries: 3,              // 最大重试次数
};
```

### 配置缓存服务

在 `src/services/cacheService.ts` 中可以调整：

```typescript
const DEFAULT_CONFIG: CacheConfig = {
  defaultTTL: 1000 * 60 * 60 * 24,  // 24 小时过期
  maxSize: 50 * 1024 * 1024,        // 最大 50MB
};
```

### 更换 AI 模型

如果你想使用其他兼容 OpenAI 格式的 API：

1. 修改 `src/services/ragService.ts` 中的 `DEEPSEEK_API_URL`
2. 修改模型名称（如 `deepseek-chat` → `gpt-4`）
3. 更新 `.env.local` 中的 API Key
